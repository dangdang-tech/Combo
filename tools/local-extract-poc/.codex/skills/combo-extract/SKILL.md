# Combo 能力项提取

你要把用户指定的本机会话历史归纳成可复用的能力项，并逐项提交给 combo CLI。

## 硬约束（任何情况下不可违反）

- 会话原文不上传、不外发：你只提交归纳后的能力项 JSON。
- 提交只走 `node {POC_DIR}/combo.mjs push`，不使用任何其他上传方式。
- 产出里不得包含密钥、邮箱、IP、内网地址等敏感信息（push 会机器扫描并拒收）。

## 工作流程

1. 先运行 `node {POC_DIR}/combo.mjs spec` 获取当前提取规范，通读并严格照办——规范会随服务端更新，以每次拉到的为准。
2. 按规范逐个分析用户指定的会话文件，归纳能力项。
3. 每个能力项写成单个 JSON 对象临时文件，立即 push；报错按错误提示修正后重试。
4. 完成后向用户汇报：入库清单（名字 + 一句话）与放弃的候选及原因。

`{POC_DIR}` 指本 skill 所在项目根目录（即包含 combo.mjs 的目录）。
