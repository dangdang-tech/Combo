# Agora 怎么评测 Agent / Mini App

> 本文融合本地 harness/self-verification 研究 + Agora 运营级评测设计

> 一句话立场：**我们的评测不是给 Agent「打个分」，而是从 harness engineering / self-verification 研究里长出来的一套确定性优先的评分协议。**

---

## 一、为什么这么评：一条 self-verification 主线

评测体系的工程决定不是拍脑袋，而是从一条研究主线推出来的——**confident praising（生成态只夸不挑）是系统性偏差**，破它要靠机制而非更强的模型。

- **评测 = Feedback 控制，且 Computational 优先于 Inferential。** Harness 框架里，canonical test 属于 **Feedback + Computational 象限**（执行后跑 test、对 expected output），CLAUDE.md 式约束属于 Feedforward + advisory 象限——两类互补，但判分主力放在能确定性执行的那一格。→ **客观层主力是确定性 canonical test，LLM judge 只兜底无法 verifier 化的开放产出。**
- **Generator/Evaluator 分离 + Sprint Contract，是同一套 self-verification 的两面。** 谁来评：模型在生成态会 confident praising，所以走 GAN 式分离（不同 Agent / session / model），能力分绝不让 Agent 自评、也不裸用单个 LLM judge（单 judge 本质仍是「自己人评自己人」）。什么时候定标准：验收测试在执行前协商写定（Sprint Contract）。→ **canonical test 是上架前就写好的验收契约，杜绝「mark as complete 但跳过边界」。**

> Permission（越权调工具 / 访问文件）同源于「确定性必定执行」原则：得像 Hooks 那样每次必跑，而非靠模型「记住」——这条直接落到下表 Permission 行的判分性质。

---

## 二、三层评测引擎：评什么 + 怎么评

| 层 | 用户问题 | 怎么评（落到 canonical test） | 判分性质 |
|---|---|---|---|
| **客观能力** | 能不能做出来 | **Smoke + Capability + Permission** 自动 test suite；能力分用 **QYI**（Quality-Yield Index）表达。Permission 同走确定性判分，像 Hooks 一样必定执行 | 确定性、亚秒级 pass/fail，**上线即出** |
| **主观体验** | 我喜不喜欢 | 真实调用后的加权用户评分（满意度 / 停留 / 复用），复用 Z7 抗刷分权重函数 | 主观，需真实样本累积 |
| **长期可信** | 长期靠不靠谱 | Regression（退化）+ Refund evidence（失败归因）+ 复购 / 退款率时间序列 | 时间序列，周-月级 |

**关键设计：三类分解耦、各有出分节奏，绝不揉成一个综合分**——硬揉是评分失真的头号原因。

> QYI 自带区分度：HeuriGym（ICLR 2026、CSO 亲手做、9 个 SOTA 验证）上专家基线 = 1.0，而 GPT-o4-mini-high / Gemini-2.5-Pro 这类 SOTA 也只到 0.6。

---

## 三、怎么保准：confident praising → 三道防线 + 贝叶斯置信阈值

**承接主线**：单 judge 逃不开偏差、易被钓、易翻盘，所以开放产出的 LLM 判分强制走三道防线：

| 防线 | 做法 | 关键数字 |
|---|---|---|
| **① Pairwise + 强制 swap** | 两序都判、不一致记平局 | GPT-4 交换 A/B 顺序后一致率仅 **65%**（约 1/3 翻盘），位置偏差可造成 **10-15 个百分点**胜率漂移 |
| **② PoLL 异构 3 家投票** | Claude / GPT / 开源各一 | 成本仅 **1/7-1/8**，κ 全面超单一 GPT-4（NQ 0.627→0.763），方差降一半（标准差 6.1→2.2） |
| **③ 注入清洗** | 把创作者自报话术当不可信输入隔离 | 不做则注入 **ASR>30%**（System-Prompt 类 73.8%、自适应越狱 98.9%），capability 分被钓爆 |

**置信度用统计兜底，把样本量写进分数**（少交互不敢给高分）：

- **冷启动不留白、也不虚高**：T+0 用 QYI 客观先验分 + IMDb 式贝叶斯收缩（先验强度 m≈10-50），少交互拉回全局均值。
- **暂定分 → 确证分翻转**：**N≥100 且 Wilson 95% 半宽 < ±0.05** 才转「确证分」，正式参与排序与对外传播；不足则标「评测中·低置信」。
- **judge 自身也要被评**：上线前用 ≥200 条 golden set 标定，judge-human **κ≥0.7-0.8** 才放行，持续监控 swap 一致率 / κ 漂移。

## 四、多久出分

| 维度 | T+0 上线 | T+天（~10-30 交互） | T+周（~100-300 交互） | T+月 |
|---|---|---|---|---|
| **客观能力分（QYI）** | ✅ **上线即确证**（确定性判分，不靠样本量） | 稳定 | 稳定 | 稳定 |
| **主观体验分** | 仅贝叶斯先验占位（评测中·低置信） | 初步可信分（脱离先验主导） | ✅ **翻转为确证分**，正式参与排序 | 持续校准 |
| **长期可信分** | 待观察 | 早期退款 / regression 基线 | 退款率趋势可读，>20% 触发暂停 | ✅ 补齐长期分 |

> 上线瞬间即有「客观分（确证）+ 主观分（评测中）」；**T+周（约 100-300 次有效调用）主观分翻转为确证分**。

## 五、为什么是护城河

评测准确性是**供给侧存亡开关**：KOL 推荐天然占社交信用优势（早期 80-90% 购买来自 KOL），腰部创作者唯一翻盘通道，就是一个用户真信、能把高分翻译成调用量的评分系统。而它的公信力恰恰来自那条 self-verification 内核——**评测系统一旦迎合创作者，本质就是 confident praising 的系统级版本**（Generator 自夸的同构）：所以专业客观性不是姿态，是「高分可被翻译成调用量 → 腰部被发现概率 > 0」的产品前提。

大厂短期复制不了，原因有二。其一是乘积壁垒：**数据飞轮（真实消费链路 × 科学评测两端打通）× 方法学（HeuriGym/QYI 原生资产）× 时间积淀（golden set、贝叶斯调参、Elo 票数）**三者相乘，缺一不可。其二更关键——**结构性互斥**：大厂做「独立第三方裁判」等于给自家旗舰模型当客观裁判，与订阅商业模式天然冲突。中立第三方评测的位置，结构上不会被有自有生态的大厂占走。
