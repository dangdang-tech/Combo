# 复盘 · 「连接本机」零选择导入

> 用户问:「为什么一定要目录授权,不能帮用户选吗?」→ 纯网页不能(安全边界),
> 但代码跑在用户本机就能。本篇是这条路的落地。

## 核心结论
「网页帮用户选目录」= 「网页能背着用户读盘」,被浏览器从根上禁止。要「零选择 / 我们帮他搞定」,
唯一前提是**代码跑在用户自己机器上**(用其系统权限直接读 `~/.claude`、`~/.codex`,像 `ls` 一样不需授权)。

## 形态:本机助手 + 一次性配对码
1. 网页(创作者已解锁)点「🔌 连接本机」→ `POST /api/connect/new` 铸一个 6 位**一次性配对码**。
2. 网页显示一行命令:`curl -fsSL <BASE>/connect.mjs | node - <配对码>`(+ 复制按钮)。
3. 用户终端粘贴 → 服务器吐出 `connect-helper.mjs`(注入了 BASE)→ 本机 Node 运行:
   扫 `~/.claude/projects` + `~/.codex/sessions` → 提取精简文本 → `POST /api/connect/upload`(凭码)。
4. 网页每 2s 轮询 `/api/connect/poll?code` → 落地后拿到 app id + stats → **自动进下一步**。

## 关键设计 / 踩坑
| 点 | 处理 |
|---|---|
| 助手没有登录 cookie | `/connect.mjs`(脚本公开)与 `/api/connect/upload`(凭一次性码)在访问码闸**放行**;铸码 `/new` 与轮询 `/poll` 仍需创作者解锁 → 只有已解锁的人能发起 |
| 助手上传 `fetch failed / bad port` | 本机 node 的 `fetch`(undici)有代理/“bad port”怪癖 → 改用 **`node:http/https` 直发**,避开 undici |
| BASE 怎么传给脚本 | 服务器按请求 `Host` + `x-forwarded-proto` 注入 `__BASE__`(railway 给 https)|
| 配对码安全 | 6 位随机、20 分钟 GC、铸码需解锁;单次用完即映射到 app |
| 隐私 | 助手只上传提取后的文本(标题+条数+前 ~12 条用户消息),原始日志不出本机 |
| 复用 | `buildUploadedApp()` 抽出,`import-uploaded` 与 `connect/upload` 共用 |

## 测试(全绿)
- 本地真实链路:助手扫 **645 段(Claude 524 + Codex 121)** → 上传 → `poll ready`,271 段去重、11823 消息 ✓
- 线上:`/connect.mjs` 注入 https BASE ✓、`/new` 未解锁 401 ✓、`/connect.mjs` 公开 200 ✓
- 线上 UI:点「连接本机」显示带 https URL + 配对码的命令 + 复制;模拟上传后**网页轮询自动跳到导入完成**(3 段)✓;零 console 错误 ✓
- 三文件语法 ✓

## 与「一键授权(FS Access)」的关系
并存,各取所需:
- **一键授权**:纯网页内,Chrome/Edge,选一次目录。
- **连接本机**:任何浏览器,**零目录选择**,代价是终端粘一行命令(开发者日常)。

## 还可做
- 助手支持 opencode(SQLite,需本机 sqlite3)。
- 公司代理环境:助手 `node:https` 不自动走代理,可加 `HTTPS_PROXY` 支持。
- 把命令做成 `npx agora-connect <码>`(发 npm 包)更短。

## 一句话
「帮用户选」的本质是「让我们的代码替他读盘」——这只能发生在他自己机器上。本机助手 + 一次性配对码,
让用户在终端粘一行、零目录选择,网页自动接上。安全闸完好(脚本公开、上传凭码、铸码需解锁)。
