## 打磨的根:消除三处重复的解析逻辑
Claude/Codex 的 jsonl 解析此前**重复在三处**(loop.html 客户端、connect-helper 本机助手、loop-server 服务端 scan),会**漂移**,且**零测试覆盖**——而解析是导入正确性的命脉(误解析则下游全错)。

收敛为一个纯模块 **`parse-sessions.mjs`**(唯一真源):
- `parseClaude` / `parseCodex` / `parseSession`(自动识别)/ `isCodexFormat` / `codexText` / `baseName` / `dedupeAndStats` / `sessionKey`
- 纯函数,无 DOM / fs / node 依赖 → 可直接单测

**三处共用同一真源**:
- 服务端:直接 `import`(scanClaude/scanCodex/buildUploadedApp)。
- loop.html(classic script)+ connect-helper(stdin ESM):服务器**注入**——读 parse-sessions.mjs、去掉 `export`、替换 `/*__PARSERS__*/` 占位。永不漂移。
