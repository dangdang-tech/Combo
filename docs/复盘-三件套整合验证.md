## 2. 端到端冒烟(真 HTTP + 真 SSE,多经验体注册表)

启动:`EXP_BRAIN=mock EXP_PORT=7820 node experiments/exp-server.mjs`
注册表实测扫出三个经验体:`career, extracted, redliner`(缺省 career)。

对 career 与 redliner 各跑一轮 collaborator(`POST /api/session` → `GET /api/events` SSE → `POST /api/run` → 收 `task.completed`):

```
[career]   kind=artifact rows=3 warnings=[] citationIssues=[] anyWhy=true anyCite=true
[redliner] kind=artifact rows=3 warnings=[] citationIssues=[] anyWhy=true anyCite=true
VERDICT={"careerOk":true,"redlinerOk":true,"adversarialOk":true}
```

结论:同一 Engine 实例按 `body.exp` 选不同经验体建独立 session、编译不同 system prompt、SSE 推回带 `cell.why` + 真实 `citedBlockIds` + 空 `citationIssues` 的合法 artifact。**多经验体注册表 works**——换 JSON 即换垂直,服务端代码对 career / redliner 一视同仁。

> 注意(诚实交付):端到端用的是共享 career 味 mock brain,故 redliner 这一轮返回的是通用 3 行矩阵,**不是** redliner 专属的「红线等级 / 建议改法」矩阵形态。该专属形态只在 `test-redliner.mjs` 用注入的红线 mock 验证过;真 redliner 形态需真 pi 跑(已留 `set -a; . ./.env` 入口,本轮未烧钱)。本轮证明的是「注册表路由 + 引擎驱动 + SSE 契约」对第二个垂直成立,不是 redliner 内容质量。

## 3. 引用真实性对抗校验(C 透明层敢上线的命门)

构造含假 id 的 artifact:`citedBlockIds = [<真实id>, "g_ghost_999", "幻觉X"]`,过 `validateArtifactCitations`:

```
issues=[{"path":"r1/a","badIds":["g_ghost_999","幻觉X"]}]  caughtFakes=true  realNotFlagged=true
```

两个假 id 都被精确抓到并定位到 cell path,真实 id 不误伤。`task.completed` 路径上 `citationIssues` 与 `warnings`(幻觉引用行)同源派生。**结论:C 披露层引用的每一条经验出处都是可证伪的——模型编出来的引用当场被纯函数兜住,这是把「为什么」摆到台面上而不怕被打脸的前提。**

## 整体 verdict:成立

- **B 是干净可编辑矩阵(地板)**,任何人都能直接读/改/pin,不被理由噪音淹没;
- **C 是每格折叠的「为什么」(按需展开层)**,点开才给 `cell.why` + 经验标签(守则/品味/案例)+ 命中守则高亮 + 幻觉引用 ⚠;
- 二者共用同一 Engine 契约,经 SSE 实时落地,引用真实性有纯函数当场兜底。

### 对用户纠结的一句话判断
> 「B 地板 + C 按需」**解决了**「全是 C 太限制」的问题:把理由从"强制阅读的正文"降级为"按需展开的脚注",决策者默认看到的是可操作的矩阵(低认知负荷),只在不信任某一格时才下钻到出处与守则。透明不再以牺牲可用性为代价——这正是与同事 No-LLM demo(漂亮但无真实出处、无优先级仲裁)的差异化落点。

## 残留风险

1. **redliner 内容质量未经真 pi 验证**:端到端走的是 career 味 mock,redliner 专属矩阵形态仅靠注入 mock 单测覆盖。上线前需真 pi 跑一轮 redliner,确认红线等级/改法的实际判断质量。
2. **提取产物 evidenceRef 行号真实性**:`extract-experience` 的行号由模型自填,normalize 只在缺失时占位,不校验已填行号是否真在 session 中存在。建议加一道轻校验(行号 ∈ session 实际行集)。
3. **citeIssues 全量重置语义**:collab.html 的 citeIssues 按 `task.completed` 全量重置,本地 pin 走 patch() 不重置(沿用上轮校验)。符合预期但需在文档明确,避免误判 pin 后引用状态。
4. **两份 html 渲染栈重复**:collab.html 与 redliner.html 刻意复制改造,渲染/编辑代码重复。若第三个垂直出现,应抽公共渲染层,否则维护成本线性增长。
5. **exp-server 对 redliner.html 的静态路由是逐个手加**:无通配 fallback,每加一个垂直页面要补一行路由。
