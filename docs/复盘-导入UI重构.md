# 复盘 · 导入 UI 重构(两卡 + 引导 + 三列 + Personal Context + 加宽)

> 用户反馈:按钮太多;一键导入要选文件夹但不知道选哪个、易出错;萃取结果一栏展示不下;
> 要 Personal Context 展示位;UI 太小;失败态奇怪、要更稳。

## 改了什么
| 反馈 | 改动 |
|---|---|
| 只留两个按钮 | 导入界面 = 两张大卡:**一键导入本地历史** / **通过 CURL 命令导入**;删掉粘贴栏、选文件夹按钮、复访自动加载 |
| 不知道选哪个文件夹 | 一键导入改为**引导页**:三步,明确「按 ⌘⇧G → 粘 `~/.claude/projects`(带复制)→ 打开」,并警告别选主目录(Chrome 报 "system files");导完引导再导 `~/.codex/sessions`;附「改用 CURL」出口 |
| 一栏展示不下 | 萃取能力项改为**一行三列网格**(`.candgrid`),一屏看完 |
| Personal Context 展示位 | 新增 **Personal Context 面板**:你反复在做(能力主题)/ 主要项目(候选证据聚合)/ 规模·工具·跨度(导入统计派生)——无额外 LLM、不编数字 |
| UI 太小 | `.wrap` 1000→**1280px** + padding 加大;新增 impcard/guide/pctx/candgrid 卡片化样式 |
| 失败态奇怪 / 要稳 | 所有导入失败统一走 `importLoader.fail`(标红 + 「重新开始」→step1);引导页有「返回」「切 CURL」出口;解析器 83 条单测兜底;状态持久卷保留 |

## 一个待确认
用户原话第 (a) 项是「一键导入本地**截图**」。整条产品是解析 Claude/Codex 的 jsonl **对话历史**,无截图功能,判断是口误/转写错误,按「一键导入本地**历史**」实现。若确需截图(OCR)导入,是另一条新管线,待用户确认。

## 验证
- 单元 83/83;两卡布局、引导页、三列+Personal Context 截图均达标
- 注入前后 JS 语法有效;线上引导页路径复制/打开/返回/切CURL 出口齐全;零 console 错误

## 一句话
导入从「五个入口 + 选文件夹靠猜」收敛成「两张卡 + 手把手引导」;萃取从「一长列」变「三列 + 你是谁的画像」;整体加宽。该解决的友好性与展示问题都落地了。
