# Agora 怎么评测 Agent / Mini App（通俗讲解版）

> 这份文档回答一个问题：**用户做了一个 Agent（我们对外叫 Mini App），Agora 怎么给它"评分"，这个分凭什么可信、多久能出来。**
> 全文尽量说人话。每个黑话术语都会先用一句话 + 一个比方解释清楚；每个核心方法都给出论文出处和原文数字，方便你核对。

---

## 0. 先把黑话翻译成人话（术语速查）

读正文前，先认识几个词。后面看到就不懵了。

| 术语 | 一句话解释 | 打个比方 |
|---|---|---|
| **Agent / Mini App** | 一个能自己干完一件活的 AI 程序（比如"写小红书脚本"）。对用户叫 Mini App，对工程师叫 Agent。 | 一台自动贩卖机：投钱 → 出货 |
| **评测 / 评分** | 给这台贩卖机打个质量分，让用户知道值不值得买 | 大众点评的星级 |
| **canonical test（标准测试）** | 平台预先写好的一套标准考题，机器自动跑、自动判对错 | 驾照考试的标准科目，不靠考官心情 |
| **确定性判分 vs AI 打分** | 确定性 = 跑一下看结果对不对（铁板钉钉）；AI 打分 = 让另一个大模型主观评好坏（会有偏见） | 确定性 = 称体重；AI 打分 = 评委打印象分 |
| **LLM-as-judge（大模型当评委）** | 用一个大模型去给另一个 AI 的输出打分 | 让 GPT 当作文阅卷老师 |
| **confident praising（自信吹捧）** | 大模型评价自己刚写的东西时，倾向只夸不挑毛病 | 自己改自己的作文，总觉得写得不错 |
| **QYI（质量-产出指数）** | 一个把"做没做成"和"做得多好"合成一个数的能力分公式 | 考试既看及格率，又看平均分 |
| **PoLL（评委团）** | 不用一个评委，用好几家不同的大模型一起投票 | 体操比赛 5 个裁判去掉最高最低分 |
| **prompt injection（提示注入）** | 创作者在 Agent 里藏话术，把 AI 评委"忽悠"出高分 | 考生在卷子上写"老师您辛苦了给个满分吧" |
| **ASR（攻击成功率）** | 上面这种忽悠，有多大比例真的得逞 | 作弊成功的概率 |
| **κ / Cohen's kappa（一致性系数）** | 衡量"AI 评委"和"人类评委"打分有多一致，扣掉了瞎蒙也能对上的部分。0=瞎蒙，1=完全一致 | 两个老师改同一批卷子，给分一不一样 |
| **贝叶斯收缩（Bayesian shrinkage）** | 评价人数少时，别急着给高分/低分，先往全站平均分上拉一拉 | 只有 1 个人评 5 星的小店，不该排在 1000 人评 4.8 的店前面 |
| **Wilson 置信区间下界** | 用"有 95% 把握，真实好评率至少是多少"来排序，对小样本天然保守 | 宁可低估，不冤枉，也不虚高 |
| **Elo / Bradley-Terry** | 让同类 Agent 两两 PK，按胜负算出排行榜分数 | 象棋天梯分 |

## 1. 一切的根：为什么"评分"这么难？——因为 AI 会"自信地吹捧自己"

我们做评测的所有设计，都是从一条研究结论倒推出来的，这条结论来自我们本地对 Agent 可靠性的研究：

> **大模型在"生成"东西的时候非常自信，但这种自信和"对不对"几乎没关系。你让它评价自己刚写的代码，它会"confident praising"——只列优点、忽略问题。这不是它在说谎，而是"生成模式"和"评估模式"之间天生有系统性偏差。**
> —— 这一点 Anthropic 称为"提升 Agent 可靠性**最高杠杆的一件事**"（Self-Verification）。

这条结论一旦成立，三件事就被锁死了（它们其实是同一个道理的三个面）：

**①（谁来评）不能让 Agent 给自己打分，也不能只用一个 AI 当评委。**
因为"自己评自己"必然吹捧。解法叫 **GAN 式分离**（GAN 是 AI 里"生成器 vs 鉴别器"对抗的经典思路）：把"干活的 Agent（Generator）"和"打分的评委（Evaluator）"彻底拆开——不同程序、不同会话、甚至不同模型。
> ⚠️ 注意：哪怕换成"一个外部的大模型评委"，本质还是"AI 评 AI"，单个评委照样有偏见（见第 3 节）。所以光分离不够，还得给评委上三道保险。

**②（什么时候定标准）考题要在 Agent 上架之前就定死。**
这叫 **Sprint Contract（冲刺契约）**：干活的和评分的，在开工前就把"什么算合格"写成可执行的测试用例。这样就杜绝了 AI 常见的"我觉得我做完了"——其实跳过了一堆边界情况。
> 落到 Agora，就是：**canonical test（标准测试）= Agent 上架前就写好、双方认可的验收合同。**

**③（凭什么可信）评分系统自己绝不能"迎合创作者"。**
这跟 confident praising 是同一个病根——**一个迎合创作者的评分系统，本质就是"自信吹捧"的平台级放大版**。所以"专业客观"不是态度问题，而是整个商业模式的前提（详见第 6 节：分准了，腰部创作者才有出头的可能）。

**这一节的出处：**
- 本地研究：BENZEMA 知识库 `self-verification` / `harness-engineering` / `creative-agent-design`（蒸馏自 harness-engineering 深度研究报告）。
- 框架来源：**Harness Engineering 四象限** —— 把控制手段分成"前馈/反馈 × 确定性/推理性"。评测属于"反馈控制"，且**确定性（跑测试）应优先于推理性（AI 打分）**。原话：先跑确定性 tests，再用 LLM 做高层逻辑审查。

## 2. 怎么评：三层引擎，各打各的分，绝不揉成一个数

我们把一个 Agent 拆成三个维度分别评，因为它们"能不能快速出分"完全不同：

| 层 | 用户其实在问 | 具体怎么评 | 判分方式 | 出分速度 |
|---|---|---|---|---|
| **① 客观能力** | "它到底能不能干成这活？" | **Smoke / Capability / Permission** 三类标准测试自动跑；能力分用 **QYI** 表达 | **确定性**（跑一下看结果），不靠 AI 主观 | 上线**几分钟内**就有 |
| **② 主观体验** | "我用着喜不喜欢？" | 真实用户用完后的加权评分（满意度、停留时长、有没有再用） | 主观，必须等真实用户攒够样本 | 要**几天到几周** |
| **③ 长期可信** | "它长期靠不靠谱？" | 更新后有没有退化、退款率、复购率随时间的曲线 | 时间序列 | 要**几周到一个月** |

> **关键设计：这三个分各算各的，绝对不揉成一个"综合分"。** 把上线即出的能力分，和要攒几周的体验分硬平均成一个数，是评分系统失真的头号原因。

**这三类考题分别考什么（canonical test 详解）：**
- **Smoke test（冒烟测试）**：最基础——这个 Agent 在用户环境里**能不能正常启动、找得到它要用的工具和资源**。名字来自硬件行业"通上电冒不冒烟"。
- **Capability test（能力测试）**：给一道标准任务，**能不能产出可接受的结果**。这是核心，用 QYI 打分（见下）。
- **Permission test（权限测试）**：它有没有**偷偷调用不该调的工具、访问不该碰的文件**。这一类**必须用确定性检查、每次必跑**——就像代码里的安全钩子（hook），不能指望 AI"自己记得别越权"。

## 3. 核心方法详解（一）：能力分怎么打——QYI

**【是什么】** QYI = Quality-Yield Index（质量-产出指数），把两个东西合成一个能力分：
- **Yield（产出率）**：一批标准任务里，**做成了几道**（通过验证的比例，类似及格率）。
- **Quality（质量）**：做成的那些里，**做得多接近专家最优解**（你的答案成本 ÷ 专家标杆答案成本，封顶 1）。
- 两者取**加权调和平均**——意思是：只刷及格率但质量差、或质量高但大部分做不成，都会被拉低分。不能偏科。

**【打个比方】** 像评价一个外卖骑手：既看**送达率**（产出），又看**每单比最优路线慢多少**（质量）。光送得多但单单超时，照样低分。

**【为什么用它，凭什么有区分度】** 这把尺子不是我们自造的——它来自团队 CSO 亲手做的学术工作 **HeuriGym**，在 9 个当前最强模型上验证过：
- 人类专家基线 QYI = **1.0**；
- 连 GPT-o4-mini-high、Gemini-2.5-Pro 这种顶级模型也只到 **0.6**。
- 说明这把尺子**有真实区分度**，不是人人满分的橡皮尺。

**【整个能力评测怎么自动跑】** HeuriGym 的流程是一个闭环：**AI 提出方案 → 代码自动执行拿到反馈 → 再改进**，每道题自带两个程序——**verifier（验约束，看有没有违规）+ evaluator（算目标，看成本多少）**。**全自动、不需要人、可重复算**。这就是为什么客观能力分能"上线几分钟内出"——判分逻辑本身是秒级的（慢的只是被测 Agent 自己跑推理，所以我们只跑"小而精"的核心题集，不跑全量）。

**【出处】**
- HeuriGym 论文：arXiv **2506.07972**（ICLR 2026），项目页 cornell-zhang.github.io/heurigym，GitHub cornell-zhang/heurigym。
- 行业对照：因为 AI 当评委有偏见、复杂任务错误率 >50%、与专家一致性仅约 64–68%，**约 74% 的评测团队至今仍保留人工复核**——这正是我们把能力分押在"确定性可执行测试"而非"AI 打分"上的原因。

## 4. 核心方法详解（二）：非得用 AI 评委时，怎么防止它被骗——三道防线

有些东西没法用程序判对错（比如"这段文案好不好"），只能退回到"让大模型当评委"。但学术界已经反复证明：**单个 AI 评委漏洞百出**。所以我们强制上三道防线。

### 先看单个 AI 评委有多不靠谱（三大偏差，都有论文数字）

| 偏差 | 大白话 | 原文数字 | 出处 |
|---|---|---|---|
| **位置偏好（position bias）** | 同样两个答案，**谁放前面就偏向谁** | GPT-4 把 A/B 答案**调换顺序后，判断只有 65% 还一致**——约 1/3 会因为位置而翻盘；光位置就能造成 **10–15 个百分点**的胜率漂移 | MT-Bench, arXiv **2306.05685** |
| **啰嗦偏好（verbosity bias）** | **偏爱长答案**，哪怕是废话堆砌 | 用"重复列表"灌水去骗，GPT-3.5/Claude-v1 中招率高达 **91.3%**（新模型已基本免疫） | 同上 / arXiv 2410.02736 |
| **自家偏好（self-enhancement）** | **偏爱自己家模型**的输出 | 早期估计 GPT-4 给自己 +10%、Claude +25%（新模型已显著减小） | 同上 |

### 三道防线（每道都有出处）

**① 强制 swap（换位再判一次）**
同一对答案，**正反两个顺序都判一遍，结果不一致就记平局**。这是消除"位置偏好"的标准做法。
> 出处：MT-Bench（arXiv 2306.05685）。

**② PoLL：评委团投票（性价比最高的一招）**
不用一个评委，用**三家不同的大模型**（比如 Claude + GPT + 一个开源模型）一起投票。好处量化得很漂亮：
- 与人类的一致性（κ）**全面超过单个 GPT-4**：自然问答任务 **0.627 → 0.763**；
- 打分**波动减半**（标准差 6.1 → 2.2）；
- **成本反而只要 1/7~1/8**（三个小模型比一个大模型便宜）；
- 还能把"被忽悠"的攻击成功率**降低 20~47 个百分点**。
> 出处：**"Replacing Judges with Juries"（PoLL）**, arXiv **2404.18796**；对抗鲁棒性 arXiv 2504.18333。

**③ 注入清洗（把创作者的自夸当"可疑输入"先洗掉）**
创作者**真的能**在 Agent 里藏话术骗高分，这叫 prompt injection。不防的话有多严重：
- 优化过的注入，**攻击成功率（ASR）> 30%**；
- "System-Prompt"类攻击成功率 **73.8%**；
- 自适应越狱在特定场景高达 **98.9%**，还能绕过 **67.5%** 的单一防御。
所以必须把创作者自报的话术当作不可信输入，做隔离和注入检测，否则能力分会被直接"钓爆"。
> 出处：优化型注入 arXiv **2403.17710**；对抗攻击 arXiv **2504.18333**。

### 评委自己也要被考核（防止评委跑偏）

**【方法】** 上线前，先让人工标注一批"标准答案卷"（**golden set，≥200 条**），再算 AI 评委和人类的一致性 **κ**，**κ ≥ 0.7~0.8 才允许这个评委上岗**。之后持续监控它的 swap 一致率、κ 有没有漂移。
**【一个反直觉的坑】** 光看"相关性高"不够：曾有 36 个模型相关性达标（r≥0.88），但**只有 27 个**真正达到人类级一致性（κ）。相关 ≠ 一致。
> 出处：**"Judge's Verdict"**, arXiv **2510.09738**。

**【结论】** AI 评委在**可核验**的任务上（任务完没完成、代码跑没跑通）能做到 **κ 0.85~0.95、与人类一致率 85%+**，达到甚至超过人类裁判之间的一致性；但在**纯主观**好坏上只能做到中等——所以我们把能力分主力放在确定性测试，AI 评委只兜底开放产出。

## 5. 核心方法详解（三）：分数准不准、要多少人评才算数——置信度

打分要面对一个老问题：**只有 3 个人评的 5 星店，凭什么排在 1000 人评的 4.8 星店前面？** 我们用两个统计工具解决。

### ① 贝叶斯收缩：人少时，先把分往全站平均拉一拉

**【方法】** 用 IMDb 那套经典加权公式（大白话版）：
> **最终分 = （本店票数 ÷(本店票数+m)）× 本店平均分　＋　（m ÷(本店票数+m)）× 全站平均分**

这里的 **m 是"先验强度"**，相当于先塞给这家店 m 条"值等于全站平均"的虚拟评价。
- 评价人数很少时（票数→0），分数被拉回全站平均（**不敢给虚高分**）；
- 评价人数足够多时，才收敛到它自己真实的平均分。
- m 取多大？经验值 **10~50**（IMDb 历史上用过 25000 这种量级）。

**【反例】** Steam **不做**这个，只用"好评数÷总数"，所以会出现"1 条好评的店 > 48 好评 1 差评的店"的荒谬排序——这正是我们要避免的。
> 出处：IMDb Ratings FAQ；Bayesian average（维基）；Evan Miller；反例 SteamDB rating algorithm。

### ② Wilson 置信下界：用"最保守的真实好评率"来排序

**【方法】** 不直接用好评率排序，而是算"**有 95% 把握，真实好评率至少是多少**"（统计学叫 Wilson score 区间的下界，95% 对应系数 z=1.96），用这个下界排。它对小样本天然保守——**宁可低估，绝不虚高**。
> 出处：Evan Miller《How Not To Sort By Average Rating》——这是排序界的经典做法。

### ③ "评测中"什么时候转"已确证"——给个硬阈值

样本越多，估计越准：误差范围按 **1/√n** 收窄（n 是有效评价数）。据此定规矩：
- **评测中（低置信）**：有效调用 **< 100** 次，或 95% 区间半宽还 > ±0.05 → 只显示贝叶斯收缩分，标注"评测中"。
- **已确证**：有效调用 **N ≥ 100 且 95% 区间半宽 < ±0.05** → 转正，正式参与排序和对外传播。
> 出处：二项比例区间宽度公式（arXiv 2109.12464）；样本量推导：±0.10 需约 96 票、±0.05 需约 385 票（极端情况下约 139 票）。

### ④ 同类 Agent 排行榜：用 Elo（参考大模型竞技场）

同一品类的 Agent 两两 PK，按胜负算天梯分（**Bradley-Terry**，比简单胜率更稳，打败强者加更多分）。票数不够就标"Preliminary（评测中）"。
> 出处：LMSYS Chatbot Arena 方法（lmsys.org/blog/2023-12-07）。参考值：大模型竞技场进榜约需 ≥5000 票；我们内部 5 个同类 Agent（10 个配对）约 **1500~3000 票**就能让前三名置信区间收窄到 ±15 Elo 以内。

## 6. 多久能出分？（你最关心的问题）

| 维度 | T+0 上线瞬间 | T+几天（约 10~30 次真实使用） | T+一两周（约 100~300 次） | T+一个月 |
|---|---|---|---|---|
| **客观能力分（QYI）** | ✅ **当场出，且当场就是确证分**（确定性判分，不靠人数） | 已稳定 | 已稳定 | 已稳定 |
| **主观体验分** | 只有贝叶斯先验占位，标"评测中·低置信" | 出"初步可信分"（开始脱离先验） | ✅ **转为确证分**，正式参与排序与对外传播 | 持续校准 |
| **长期可信分** | 暂无（标"待观察"） | 出早期退款/退化基线 | 退款率趋势可读，>20% 触发暂停整改 | ✅ 补齐 |

**一句话总结：**
> **一个 Mini App 上线的那一刻，就立刻有"客观能力分"（确定性、当场可信）；几天后有"初步体验分"；大约一两周、积累 100~300 次真实有效使用后，体验分转为"确证分"——这就是它能正式上榜、能跟 KOL 推荐掰手腕的时间点；一个月左右补齐长期分。**

## 7. 为什么这套东西是护城河（大厂为什么短期抄不走）

**第一，评测准不准，是 Agora 的"生死开关"，不是锦上添花。**
早期 80~90% 的购买都来自 KOL 推荐（KOL 自带社交信用）。腰部、长尾创作者唯一的翻盘机会，就是**一个用户真信、能把"高分"翻译成"真实调用量"的评分系统**。评测准 → 出现几个"高分小创作者的调用量超过 KOL"的案例 → 大家开始信这套分 → 长尾持续进场 → 供给端飞轮转起来。评测一旦不准，这条路就堵死，平台只剩 KOL，产品就死。
而它的可信度，恰恰来自第 1 节那条内核——**评测系统一旦迎合创作者，本质就是 confident praising 的平台级版本**。所以"专业客观"是产品前提，不是姿态。

**第二，这是三样东西的乘积，缺一不可：**
- **数据飞轮**：标准考题、标准答案卷（golden set）、失败案例库，全都要从真实创作者的"脏数据"里反复打磨。大模型竞技场只有投票没有交易链路、学术评测没有真实反馈、GPT Store 的数据归 OpenAI 不外流——**只有 Agora 能把"真实消费"和"科学评测"两端打通**。
- **方法学**：HeuriGym / QYI 是 CSO 亲手做、ICLR 2026、9 个顶级模型验证过的原生资产，不是买来抄来的。
- **时间积淀**：评委的 κ 校准、贝叶斯参数调优、Elo 票数累积，全都要真实流量慢慢喂。

**第三，也是最硬的一条——结构性互斥：**
大厂做"独立第三方评分"，等于要**给自己家的旗舰模型当客观裁判**，这跟它卖订阅的商业模式天生打架。所以"中立第三方评测"这个位置，结构上就不会被有自家生态的大厂占走。
