# 能力关系网络 · 可视化逻辑(开发 spec,非用户可见)

> 这是给开发的实现规约,不在产品 UI 里展示(用户那条说明面板已撤)。落代码(把萃取结果渲染成 SVG/图)时照这个做。

## 它是什么
一张 **entity-relationship 图**(对齐 mem0ᵍ / Zep 的"实体作节点、关系作边"范式):
- **实体 = 能力**(萃取出的"你反复在做的事")
- **关系 = 共享同一项目**
- 中心 "你" 节点是视觉锚,不承载数据。

## 编码规则

### (a) 圆的大小 = 出现频次(段数)
- 面积 ∝ 频次,所以半径走平方根:`r = 20 + 11.958·(√count − √5)`,钳制 `r ∈ [20, 44]` px。
- 实算:18段→44 / 14段→38 / 11段→33 / 9段→29 / 7段→25 / 5段→20。
- 中心"你"固定 r=30,实心 ink,不参与缩放。
- 圆越大 = 投入越多,且不被夸大(用半径∝count 会把大能力放大到平方倍)。

### (b) 连线 / 实线(两类,层级必须分明,A 重于 B)
- **A. 共享项目实线**(能力↔能力):只有真共享同一项目才连。1.5px、ember #A73718 @60%、quadratic bezier 轻弧(控制点向簇外偏 12–15px,别穿中心)。
  - 粗细预留 = 共享项目数:`1.5 + 1.0·(shared−1)`,钳制 [1.5,4];当前都只共享 1 个故恒 1.5px。
  - 不要用"段数之和"编粗细(段数已被圆大小吃掉,会双重夸大)。
- **B. 隶属放射虚线**(中心↔能力):1px、line #E1D8CA、虚线、中心端 30% 透明渐隐到节点端 0%。只表"归属",是布局骨架,视觉必须比 A 弱。
- 描边颜色 = **可打包置信**:高=ember 实线 / 中=gold / 低=ink40 虚线+neutral 填充(低置信不用红)。置信小圆点:高=success / 中=gold / 低=ink40。

### (c) 分块处理 = 按项目聚类
- cluster key = project。同项目能力在角度上排相邻、用一圈淡色凸包 halo(neutral @35%)圈成一簇,簇上缘标 `项目名 · N 项能力`。
- **只对多节点簇画 halo**;独占项目的能力是孤立节点,不画圈、不硬连线 —— 稀疏即干净,真实反映协作结构。
- 不用色块背景区分簇(会和置信色/来源色三方打架)。

### (d) 移动端(<640px)
- 图谱降级为**按段数降序的纵向列表卡**:每张 = 能力名 + 段数条形 + 置信点 + 项目 chip。放弃放射布局。
- Personal Context 卡各区垂直堆叠,全程不横向滚动。

## 边界情况约定
- **0 能力**:空状态卡(中心"你"+ 灰虚光环 + 文案"还在积累中 — 已分析 N 段,暂未识别出反复出现的能力",提示需 ≥3 段)。PC 卡正常显示。
- **1 个**:居中对称 highlight,不放射,基准尺寸不缩放。
- **2 个**:左右/上下 180° 固定对称;共享项目则连线走中心下方弧、不穿中心。
- **3–6 个**:基准态(本稿)。
- **7–8 个**:`r_max` 降一档、标签字号降一级、连线透明度降低、碰撞则引线外移。
- **>8 ≤12**:双环(Top6–8 内环大圆完整标签 / 余项外环小圆,外环间连线默认隐藏可展开)。
- **>12**:只展示 Top8 + "+N 项其他能力"标签云,或"图谱/列表"切换。
- **孤立节点**(无共享项目):正常放射、不画能力间连线、不弱化、优先排到角度间隙大处。
- **全连通单一大簇**(都同一项目):不全量两两连线,改项目环/底色域 + hover 高亮。
- **project=null**:节点照常渲染、所属位显示"未归类"、不连线、空值保护不抛错。
- **段数并列**:同尺寸 + 稳定次级排序键(置信高>中>低 → 名称字典序)防抖。
- **能力名超长**:两行封顶 + … + tooltip,节点尺寸不变,长标签节点预留更大角度间隔或引线外移。
- **连线交叉**:一律走节点外侧弧线绕开中心、默认低透明、hover 高亮相关线。

## 选型依据
否掉 force-directed 实时物理(6 节点会挤成一坨/甩飞、不可复刻、每次渲染不一致)与继续手摆坐标。选**确定性极坐标 hub-and-spoke + 按项目聚扇区 + 按置信分环**:坐标用代码一次算好(可复现、静态稿可像素级复刻),同项目排相邻扇区→零连线交叉。参考:Obsidian Graph View(单色靠明度做层级)、nivo Network、D3 radial 示例、Cambridge Intelligence 布局选型指南。
